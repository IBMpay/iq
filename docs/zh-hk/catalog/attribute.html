屬性

此功能為了使某些產品可以作出一個比較，從而令顧客可以更了解產品的優點及不足。

例子：當顧客去買一台智能手機時，一般都會將選兩款或三款手機作出一個簡單的比較，比較的項目主像是它們的螢幕尺吋、記憶體容量、電池大小及其使用時間、上網速度等等。

使用者可以建立自己產品所需的屬性類別及項目於產品中，詳情請參閱屬性標籤。

同一類別的產品中，建議新增共同屬性使比較的資訊可以更一致及清晰。例如，如果商店銷售智能手機，則可以創建「通訊連接」屬性，此屬性需要一致地添加到商店中的所有智能手機類別的產品中。


屬性群組

屬性群組包含可以在產品比較下進行比較的相關屬性項目。例如 Clockspeed 和 No. of Cores 是隸屬 Processor 這個群組中。

OpenCart 的特性就是先從群組開始著手設定，這樣一來便不會沒有系統去放置資料，在管理或新增資料時也相對地容易找到所需的項目，所以要建立相關的屬性先由屬性群組開始。點擊上圖右上方的新增按鈕，進入下圖的頁面便會看見有兩個欄位需要填寫。

Attribute Group Name * 屬性群組的名字
Sort Order 群組次序，這個次序會影響貨品的規格表出現時的次序，並不是管理平台清單上所看見的次序。如沒有特定次序可填 0 或留空此欄。
* 為必填項目


創建屬性

前往屬性清單後從右上方的新增按鈕進入下一頁。

於創建頁面中，如下圖所示，屬性有三個欄位需要填寫：

Attribute Name * 屬性項目的名字
Attribute Group 屬性群組
Sort Order 屬性項目次序，這個次序會影響貨品的規格表出現時的次序，並不是管理平台清單上所看見的次序。如沒有特定次序可填 0 或留空此欄。
* 為必填項目


將屬性分配給產品

成功新增了屬性項目後，接著前往產品頁面去增加屬性，在產品清單隨便從一個現有貨品中點擊編輯按鈕，見清單上有鉛筆圖案藍色按鈕。前往屬性(Attributes)標籤，把剛剛建立的屬性項目加入產品中。

前往商店前台，以預設主題為例，新增了的資料會於 Specification 中顯示，如下圖所示，「New Attribute Group」分類及「New Attribute Item」都經已在該頁顯示出來。
