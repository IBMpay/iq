安裝 OpenCart

本指引將說明如何成功安裝與卸載OpenCart系統，假如您是初次接觸這類型的網站應用程式，本篇文章亦提供相關上載網頁伺服器之簡單教學。

下載並解壓縮OpenCart檔案

OpenCart（v.3.0.2.0）的最新版本可以從OpenCart網站下載（推薦一般用家）或直接從Github 平台下載（推薦網頁開發人員）。下載頁面更提供較舊版本的OpenCart方便網頁開發人員使用。 按下DOWNLOAD NOW按建，3.0.2.0版本的zip壓縮檔會直接下載到您的電腦。3.0.2.0版本的檔案對應名稱為3.0.2.0-OpenCart.zip。

如欲下載其他版本，請按Release notes 或 previous releases，查找您所需要的版本並下載。

若果您的電腦並非使用MAC OS、WINDOW 7或以上的版本，而並沒有任何解壓ZIP檔的程式，可以點擊以下網址安裝免費解壓縮程式。
  • WinRar
  • 7-Zip

完成解壓縮後便可以獲得OpenCart的系統檔案，內容包含：

upload 包含整個OpenCart系統須上傳到網頁伺服器的所有文件
license.txt 包含有關在您的網站上使用OpenCart系統的許可協議
readme.txt 檔提供了OpenCart網站上當前安裝和升級說明的連結

準備就緒後，您可以將壓縮檔中的檔案儲存到電腦上所選的位置。

上傳 OpenCart

假設您已經擁有網站伺服器，現在便開啟FTP(文件傳輸協議)程式並將與伺服器連接，將OpenCart upload 中的所有檔案上傳至伺服器內的指定位置。

admin catalog
image install
system .htaccess.txt
config-dist.php index.php
php.ini -

安裝與設定FTP客戶端

如您尚未安裝任何FTP客戶端，這裡會建議使用FileZilla，這是一個免費的FTP客戶端程式。成功連接後可以將OpenCart檔案傳輸到指定網頁伺服器內。FileZilla客戶端（不是伺服器）可以從 http://filezilla-project.org/ 下載並安裝到您的電腦上。

當啓動FileZilla後，左側視窗是電腦中的文件目錄，第一步於此視窗開啟已解壓縮的OpenCart存檔位置，然後點擊 upload；目前右側視窗仍然是空白，因為尚未連接網頁伺服器，第二步則是接上伺服器端口，成功連接後將顯示網頁伺服器的文件目錄。

連接網頁伺服器前，先從服務供應商確認及獲得以下資料：

  • 主機名 (Host name)
  • 用戶名 (Username)
  • 密碼 (Password)

新增「網站管理」

在檔案 (File)選項下，點擊網站管理 (Site Manager)後會開啟一個新視窗。點擊新網站 (New Site)並於一般(General)中填寫FTP主機名、用戶名及登入密碼，接著按下連接 (Connect)完成新增步驟。程式會按照填寫的資料自動與伺服器進行連線，成功連線後右側視窗將會顯示伺服器中網站的文件目錄。

上傳OpenCart文件

商店的上傳位置取決於您是否希望在主頁上看到商店，或是上傳於子文件夾，一個子域名等。如果你想讓 OpenCart 成為主頁面，你需要將文件上傳到網站的根目錄。

請注意，某些託管服務需要將公共文件上載到公共目錄(如public_html)，才能在網站上顯示。您應該檢查您的託管服務提供商，看看您可以上傳公共文件。

一旦確定 OpenCart 商店的上傳位置，便可於FileZilla左側視窗 upload 中的全部內容選取並拖曳到右側視窗中，或右擊選擇上傳。上傳文件可能需要花費幾分鐘到FTP客戶端上，取決於網絡及供應商的速度。

如果您希望商店位於主頁上，例如www.shopnow.com，則必須上傳 upload 的內容，而不是將資件夾上傳至伺服器中。
  • www.shopnow.com/upload
  • www.shopnow.com

cPanel

如您的服務供應商是使用 cPanel 便可以不使用 FileZilla 上傳 OpenCart 到您的網頁空間內。您只需要將 upload 內的所有文件重新壓縮一次，登入 cPanel 後再點擊 檔案管理(File Manager) 便會開啟新一頁可以作上傳或下載的網頁版FTP(文件傳輸協議)，然後上載新壓縮檔到您所需的位置 (必須於public_html內)，成功上載後，點擊右鍵選解壓 (Extract) 即可完成。

建立資料庫

接下來便要為 OpenCart 商店建立一個新的MySQL資料庫。登入伺服器管理平台並尋找 MySQL 管理 / 資料庫管理 / phpMyAdmin (不同服務供應商有不同的用字)，建立一個資料庫，同時可以制定資料庫專屬登入帳號及密碼。某些服務供應商可能需要用戶於創建資料庫時，需設有指定權限才能使用，如創建時遇上任何疑問，可向服務供應商查詢及支援。成功建立資料庫後，稍後會於安裝時使用以下資料：

  • 資料庫名稱 (Database Name)
  • 登入用戶 (Database User)
  • 登入密碼 (Database Password)

安裝程序

如以上準備功夫都完成後，開啟一個新視窗並輸入與OpenCart商店上傳位置相對的網址 (例如: www.shopnow.com)，系統便會自動跳進安裝頁面，如下圖。如果自動跳頁後顯示找不到網頁等訊息，請於FTP客戶端中確認 install 是否存在。

請跟隨以下步驟並完成安裝程序。

第一步　許可協議

閱讀許可協議後點擊 Continue 繼續

第二步　準備安裝

此頁將會檢查網站伺服器是否乎合安裝 OpenCart 的系統需求及權限。如果於清單一及二中出現了紅色提示，這表示系統需求不乎，請與服務供應商聯絡；若清單四及五顯示了紅色題示，則表示伺服器內於在指定位置缺少了任何檔案或資料夾權限不乎。

清單上所有項目都是 或 Writable 的時候 (如下圖)，便可以點擊 Continue 進行下一步的安裝程序。

由 OpenCart 2 開始，以下檔案必須更重新命名才能繼續安裝：
  • config-dist.php config.php
  • admin/config-dist.php config.php

第三步　填寫初步資料

填寫下列資料並使系統連接資料庫及建立管理者帳號。

1. 填寫資料庫訊息
Hostname 大部份服務供應商都會使用 localhost，如有疑問請向服務供應商查詢。
Username 登入用戶 (Database User)
Password 登入密碼 (Database Password)
Database 資料庫名稱 (Database Name)
Prefix 資料庫中表格的統一名稱，預設：oc_
2. 建立管理者帳戶及密碼
Username 管理者登入帳號
Password 管理者登入密碼
Email 管理者登記電郵

填寫所有資料後便可以點擊 Continue 完成最後的安裝程序。

第四步　安裝完成

成功安裝 OpenCart 後會看見下圖的頁面。這時候需要到FTP客戶端中將 install 移除才能將紅色提示框除去。此版頁有兩個選項可選擇，左方是前往OpenCart商店，右方則是進入管理者登入版面。

完成安裝後，為了提高網頁伺服器及 OpenCart 商店的安全性，我們強力建議移除 install，有關內容可於基本安全措施參閱。

安裝 vQmod / OCmod

vQmod 及 OCmod 是一種附助程式讓您新增 OpenCart 模組而毋須改動 OpenCart 系統內的編碼，詳情請參閱 vQmod / OCmod 。

卸載 OpenCart

卸載 OpenCart 只須以下步驟：

  1. 透過FTP客戶端刪除網頁伺服器內所有有關 OpenCart 的檔案及資料夾
  2. 刪除載有OpenCart系統的資料庫
當成功卸載 OpenCart 後，所有產品及客戶資料都會被刪去。在卸載前可以備份資料庫及已上存的圖片資訊，以便日後重裝 OpenCart 時快速載入所有資料。詳情可參閱備份 / 重置。
