安裝插件

預設的安裝程式

OpenCart 系統有專門的上載功能，這樣不但可以保留核心系統的文件不被破壞外，更方便使用者隨時安裝及卸載任何 OpenCart 插件。而在 OpenCart 3 的版本中，安裝功能更進一步改善了上傳方法及步驟，只要依照以下指引，使用者就可以輕輕鬆鬆安裝合適自己的功能到網店內。

安裝插件

OpenCart 3 的所有插件都必須以 *.ocmod.zip 的形式進行上傳並安裝及更新，所有的文件都會依照 儀表板(Dashboard) 中的設定，將文件放置在使用者指定的位置。假如，上傳的檔案有誤，系統會顯示錯誤訊息並無法上傳。

從管理者介面中左側的選單展開 插件(Extension)，然後點選 安裝插件 (Installer)。在安裝頁面中，點擊  上傳  後便從電腦中挑選所需的 *.ocmod.zip 然後確認。如無意外，按鈕下方的安裝進度棒會顯示為綠色。

OpenCart 3版本內，安裝頁面全面簡化，完成安裝後，新安裝的插件會在頁面的底部顯示出來；如果想卸載插件，只要點擊插件右側的  刪除  按鈕即可。

於 OpenCart 2版本，如果所上傳的插件經已存在，資料夾內的文件會於灰色框內列出，在上傳之前請先確認沒有重要文件會被覆蓋，如沒有可以按   繼續  。成功上載後，頁面會顯示綠色的訊息表示成功。

上載單一模組檔案

OpenCart 3版本內的所有插件都必須使用 *.ocmod.zip 的形式來進行上載及安裝，即便是單一模組也要放置在 *.ocmod.zip 內並命名為 index.xml 才能完成安裝程序。

如果從任何地方下載 OpenCart 的插件安裝程式，請確認上載的格式以及對應版本是否與其使用的系統相容。

於 OpenCart 2版本，如果模組只需上傳一個文件，有別於 *.ocmod.zip，單一文件的格式必須為 *.ocmod.xml 才能上載到系統中。

安裝錯誤及清除功能

此功能介紹只適用於 OpenCart 2 版本。

At times, an error could restrain the Installer from completely uploading an extension or modification file and cause it to abort the installation midway through. If this happens, the filling color in the Progress field will become red and will stop filling the bar. Right below it, you will get an error of some type. Example:

When this happens, files that had already been uploaded before the error has occurred, will need to be cleared. OpenCart has the red Clear button for that. If you had forgotten to do that, you will also get an alert on the top of the page about this every time you access your Extension Installer.

Upon clicking the Clear button, the alert should be replaced by the following "Success" message:

You should always make sure all temporary files are cleared before uploading an extension or a modification file because at the very least, they will be taking up space on your server.

設定FTP帳戶

此部份只適用於 OpenCart 2版本，在 OpenCart 3 以後的版本更簡易及快捷安裝插件。

從管理者介面左側的選單中展開 系統 (System) 選單並點擊 設定 (Setting)，從商店清單中點選   編輯  進入編輯頁面。前往 FTP 標籤 內可以看見以下欄位。

有關FTP的資料可以向伺服器供應商查詢，填好所有欄位後，請確認 Enable FTP 是啟用才儲存，否則也是無法可以安裝插件。完成這步驟後便可以到 Installer 的頁面繼續安裝。
